# 유니티 x 미디어파이프 실시간 모션캡쳐 시스템 가이드

이 가이드는 파이썬에서 추출한 동작 데이터를 유니티 캐릭터에 실시간으로 입히는 전체 과정을 설명합니다.

---

## 1. 파이썬 환경 설정 (데이터 송신측)

### 필수 라이브러리 설치
터미널(CMD 또는 PowerShell)을 열고 다음 명령어를 입력합니다.
```powershell
pip install mediapipe opencv-python
```

### 스크립트 실행
`media pipe/pose_tracker.py` 파일이 있는 폴더에서 아래 명령어를 입력합니다.
```powershell
python pose_tracker.py
```
*   **성공 확인**: 웹캠 창이 뜨고 본인의 몸 위에 랜드마크(선과 점)가 그려지며, 좌측 상단에 FPS가 표시되어야 합니다.

---

## 2. 유니티 오브젝트 구성 (데이터 수신측)

### [A] 데이터 수신 오브젝트 만들기
유니티 씬(Scene)에서 데이터를 받아줄 빈 통로를 만듭니다.
1. `Hierarchy` 창 우클릭 -> **Create Empty** 생성 (이름: `DataReceiver`)
2. 생성한 오브젝트에 `UDPReceiver.cs` 스크립트를 드래그하여 추가합니다.
   *   **Port**: `5066` (기본값) 확인.

### [B] 3D 캐릭터(모델) 설정
1. **모델 준비**: Mixamo 등에서 받은 Humanoid 캐릭터 모델을 씬에 배치합니다.
2. **애니메이터 해제**: 캐릭터의 `Animator` 컴포넌트에서 **Controller** 항목을 비웁니다 (**None**). (애니메이션이 뼈를 붙잡고 있으면 스크립트가 작동하지 않습니다.)
3. **스크립트 추가**: 캐릭터 오브젝트에 `PoseRetargeter.cs` 스크립트를 추가합니다.
4. **연결**: `PoseRetargeter` 인스펙터 창의 **Udp Receiver** 칸에 아까 만든 `DataReceiver` 오브젝트를 드래그해서 넣습니다.

---

## 3. 핵심! 10초 카운트다운 캘리브레이션

가장 중요한 것은 **T-자세**입니다.
1. 유니티 화면을 클릭한 뒤 키보드의 **'C' 키**를 누르거나 화면의 **Calibrate 버튼**을 누릅니다.
2. 로그창에 10초 카운트다운이 시작되면, 카메라 앞에서 반듯하게 **팔을 옆으로 벌린 T-자세**를 잡으세요.
3. 0초가 되면 "Calibration Complete!" 메시지와 함께 내 현재 자세가 캐릭터의 0점(기준점)으로 완벽하게 동기화됩니다.

### 주요 설정값 조절 (PoseRetargeter)
*   **Scale**: `1` (실제 미터 단위 좌표를 쓰므로 1이 가장 정확합니다.)
*   **Smoothing**: `15~20` (숫자가 높을수록 부드럽지만 반응이 느려집니다.)
*   **Hip Height Offset**: 캐릭터가 땅에 박혀 있다면 이 값을 높여주세요 (기본 `0.9`).
*   **Mirror**: 캐릭터가 나와 반대로 움직인다면 체크/해제 하세요.

## 4. 카메라 및 기타 기능
*   **카메라 조작**: `Main Camera`에 `CameraController.cs`를 넣으면 WASD와 마우스(Tab키로 전환)로 캐릭터를 자유롭게 관찰할 수 있습니다.

## 5. UDP_IP 관리
*   **UDP_IP**: 노트북(파이썬)에서 데스크탑(유니티)으로 보낼 경우, `pose_tracker.py` 상단의 `UDP_IP`를 유니티 컴퓨터의 IP 주소로 수정하세요.
*   **상황별 UDP_IP**: 
    - **장치 한대로 연결**: `127.0.0.1` (로컬호스트)
    - **인터넷 연결**: `127.0.0.1` (같은 와이파이 연결)
    - **로컬 네트워크**: `192.168.x.x` (로컬 IP) (자세한건 6.에서 설명)
    - **VPN 연결**: `VPN IP` (VPN으로 연결된 IP)

## 6. 오프라인 LAN 연결 가이드 (인터넷 없이 2대 연결)

인터넷이 없는 환경에서 LAN 케이블(Ethernet)만으로 노트북(파이썬)과 데스크탑(유니티)을 연결하는 방법입니다.

### [A] 물리적 연결 및 고정 IP 설정
1. **LAN 케이블 연결**: 두 대의 컴퓨터를 케이블로 직접 연결합니다.
2. **IP 설정 (양쪽 PC 동일하게 수행)**:
   > **참고**: 고정 IP는 원래 정해진 것이 아니라, 우리가 컴퓨터에 **직접 입력(할당)**해주는 것입니다.
   - `제어판 > 네트워크 및 공유 센터 > 어댑터 설정 변경` 열기
   - 현재 연결된 `이더넷` 항목 우클릭 -> `속성` -> `인터넷 프로토콜 버전 4(TCP/IPv4)` 선택 -> `속성` 버튼 클릭
   - **다음 IP 주소 사용** 체크 후 아래 표에 맞는 정보를 **직접 타이핑**합니다.

| 구분 | PC 1 (파이썬/노트북) | PC 2 (유니티/데스크탑) |
| :--- | :--- | :--- |
| **IP 주소** | `192.168.1.10` | `192.168.1.11` |
| **서브넷 마스크** | `255.255.255.0` | `255.255.255.0` |
| **기본 게이트웨이** | (비워둠) | (비워둠) |

### [B] 코드 수정 (파이썬)
`media pipe/pose_tracker.py` 파일의 상단 설정을 변경합니다.
```python
# 수정 전
UDP_IP = "127.0.0.1" 

# 수정 후 (유니티 PC의 고정 IP 입력)
UDP_IP = "192.168.1.11" 
```

### [C] 방화벽 확인 (중요)
데이터를 받는 쪽(**유니티 PC**)의 방화벽이 통신을 막을 수 있습니다.
- `Windows 보안 > 방화벽 및 네트워크 보호`에서 방화벽을 잠시 끄거나,
- 유니티(Unity Editor)가 개인/공용 네트워크에서 통신할 수 있도록 허용되어 있는지 확인하세요.
- **연결 확인**: 파이썬 PC에서 CMD를 열고 `ping 192.168.1.11`을 입력했을 때 응답이 오면 준비 완료입니다.

## 7. 트러블슈팅
*   **모델이 날뛰거나 꼬임**: Play 버튼을 누를 때 캐릭터가 깔끔한 T-자세였는지 확인하세요.
*   **에러 발생**: 캐릭터 모델의 Rig 설정이 **Humanoid**로 되어있는지 확인하세요. (Generic 등은 작동하지 않습니다.)
## 8. 환경 설정(Preferences) 시스템 가이드

이제 매번 스크립트를 수정할 필요 없이, 실행 중에 UI를 통해 거의 모든 설정을 변경하고 저장할 수 있습니다.

### [A] 시스템 구조
- **PreferencesManager**: 모든 설정값을 `PlayerPrefs`를 통해 컴퓨터에 영구 저장하고 관리합니다. (싱글톤)
- **MotionCaptureUI**: 사용자로부터 값을 입력받아 매니저에게 전달하고, 즉시 시스템에 반영합니다.

### [B] 유니티 에디터 설정 (최초 1회)
1. **오브젝트 생성**: 하이어라키 창에 빈 오브젝트를 만들고 이름을 `PreferencesManager`로 지정한 뒤, 동명의 스크립트를 붙여주세요.
2. **UI 연결**: `MotionCaptureUI`가 붙은 오브젝트(주로 Canvas)의 인스펙터 창에서 다음 항목들을 연결합니다.
   - **Main UI Group**: 평소 화면에 항상 떠 있는 UI 부모 오브젝트 (F11로 숨길 대상).
   - **C**: 10초 카운트다운 시작 (자세를 가다듬고 T-자세 교정 수행).
   - **Editor**: 화면의 편집 버튼을 통해 소품 생성, 맵 교체, 아바타 교체 패널을 엽니다.
   - **REC**: 녹화 시작/중지 (저장 경로는 Preferences에서 수정 가능).
   - **Preferences Panel**: 환경 설정용 팝업 창 오브젝트.
   - **InputField들**: Port, Path, Sensitivity, Move Speed 입력을 위한 필드들.
   - **Dropdown**: UI Display(보이기/숨기기) 선택용.
   - **Motion Settings**: Smoothing, Scale 슬라이더와 Mirror 토글.

### [C] 주요 기능 및 사용법
- **단축키(F11, C)**: UI 숨기기(F11)와 10초 카운트다운 캘리브레이션 시작(C)을 지원합니다.
- **포트(Port) 실시간 변경**: 포트 번호를 바꾸고 'Apply'를 누르면 UDP 서버가 자동으로 재시작되어 새 포트로 수신을 시작합니다.
- **저장 경로(Save Path)**: 녹화 파일이 저장될 절대 경로를 입력할 수 있습니다. (예: `C:\MyRecordings`)
- **감도 조절**: 마우스 회전 감도(Mouse Sens)와 이동 속도(Move Speed)를 본인에게 맞게 숫자로 세밀하게 조절하세요.

### [D] 버튼 이벤트 연결 (중요)
설정창의 **적용(Apply)** 버튼의 `OnClick` 이벤트에 `MotionCaptureUI.OnClick_ApplyPreferences` 함수를 반드시 연결해야 설정이 반영되고 저장됩니다.

## 9. 씬 에디터(Scene Editor) 사용 가이드

이제 실행 중에 맵을 바꾸거나, 새로운 오브젝트를 소환하고 직접 꾸밀 수 있습니다.

### [A] 오브젝트 조작법
1.  **선택**: 씬에 있는 오브젝트(소품)를 **마우스 왼쪽 클릭**하면 선택됩니다. 
2.  **변형**: 오브젝트가 선택되면 UI에 **Position, Rotation, Scale** 입력창이 나타납니다. 숫자를 입력하여 세밀하게 조절하세요.
3.  **복제/삭제**: UI 하단의 **Duplicate** 버튼으로 복사하거나, **Delete** 버튼으로 삭제할 수 있습니다.

### [B] 에셋 스포너 & 맵/아바타 교체
- **Spawn Asset**: 목록에 등록된 소품(의자, 탁자 등)을 카메라 앞에 소환합니다.
- **Switch Map**: 배경(스테이지) 전체를 즉시 다른 프리팹으로 교체합니다.
- **Change Avatar**: 현재 리타겟팅되고 있는 캐릭터 모델을 다른 모델로 바꿉니다.

### [C] 유니티 설정 방법
1.  `ObjectManipulator`, `AssetSpawner`, `EnvironmentManager` 스크립트를 각각 빈 오브젝트에 붙여주세요.
2.  `MotionCaptureUI` 인스펙터에서 `Preferences Panel` 슬롯과 `Editor Panel` 슬롯을 각각 연결해 줍니다.
3.  각 매니저의 `Prefabs` 배열에 소환하고 싶은 오브젝트들을 드래그해서 넣어주세요.
4.  **중요 (씬 시스템)**: `EnvironmentManager`의 `Scene Names` 배열에는 이동하고 싶은 **씬의 이름**을 정확히 입력해야 합니다.

## 10. 씬 기반 맵 시스템 설정 (중요)

이제 맵 변경 시 새로운 씬을 로드합니다. 이를 위해 다음 설정이 필수입니다.

### [A] Build Settings 등록
1.  유니티 상단 메뉴 `File > Build Settings...`를 엽니다.
2.  사용할 모든 맵 씬들을 **Scenes In Build** 목록에 드래그하여 등록합니다.
3.  등록되지 않은 씬은 코드로 불러올 수 없습니다.

### [B] 시스템 영속성 (Persistence)
- 모든 핵심 매니저(UI, 수신기, 캐릭터 등)는 `DontDestroyOnLoad`가 적용되어 씬이 바뀌어도 유지됩니다.
- **주의**: 새로 로딩되는 씬에는 카메라나 캐릭터 오브젝트가 중복으로 들어있지 않도록 비워두는 것이 좋습니다 (이미 이전 씬에서 넘어온 오브젝트들이 있기 때문입니다).
